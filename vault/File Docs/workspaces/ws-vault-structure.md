# vault/ (מבנה ה-vault)

**שייך ל:** כלל-הפרויקט

## מה הקובץ עושה
תיקיית ה-vault היא הזיכרון ארוך-הטווח של הצוות ב-Obsidian, ומנוהלת דרך הסקיל `obsidian-vault-workflow`. היא מורכבת משלושה סוגי תיעוד:

- **Meeting Notes/** — תיעוד ברמת הנושא/קומפוננטה (ארכיטקטורה, סוכנים, החלטות עיצוב). כל קובץ מכסה קבוצה לוגית קוהרנטית ולא קובץ-לקובץ. כולל קבצי נושא כמו [[agent-reuven]], [[agent-yael]], [[agent-yuval]], [[agent-chen]], [[content-pipeline]] ועוד.
- **Content Briefs/** — תיקיית editorial: briefs, תקצירים ותיאורי קמפיינים של מאמרים.
- **File Docs/** — תיעוד ברמת הקובץ הבודד. תת-התיקייה `workspaces/` (זו) מתעדת את קבצי השורש וסביבות העבודה של הסוכנים.

**התפקיד של `_index.md`:** בכל תיקייה קיים קובץ אינדקס המרכז את כל ה-topics שבה עם תיאור בשורה אחת ו-wikilinks. הוא משמש כנקודת כניסה וניווט — הסקיל קורא אותו בתחילת משימה כדי לאתר את קובץ הנושא הרלוונטי.

**Session Log:** בתוך קובץ הנושא [[project-overview]] (וקבצי נושא אחרים) מנוהל יומן סשנים — בסיום כל משימה מתווסף תקציר מתוארך (What was done / Decisions / Notes / Related) המתעד מה נעשה ומדוע. כך נשמרת היסטוריית ההחלטות לאורך זמן.
