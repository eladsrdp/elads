## מה הקובץ עושה
סקיל לפיתוח מונחה-בדיקות (TDD): כתיבת בדיקה כושלת לפני קוד המימוש, ואז כתיבת המינימום שמעביר אותה. מופעל בכל מימוש פיצ'ר או תיקון באג לפני כתיבת קוד המימוש. כולל מסמך נלווה על אנטי-דפוסים נפוצים בבדיקות.

קבצים פנימיים:
- `SKILL.md` — מחזור ה-TDD (red-green-refactor) וההנחיות ליישומו.
- `testing-anti-patterns.md` — דפוסי בדיקה בעייתיים שיש להימנע מהם.
