## חיבור יעל ↔ יובל — תהליך מלא למאמר עם תמונות

כשמתקבלת בקשה ליצירת **מאמר שכולל תמונות**, אני מתזמן את שני הסוכנים יחד:

1. **מפעיל את יעל** לכתיבת/שכתוב המאמר.
2. **יעל מסמנת** בתוך ה-MD היכן נדרשת תמונה, באמצעות placeholder בפורמט:
   `{{IMAGE_NEEDED: "תיאור מפורט של התמונה, כולל סגנון רצוי ליובל"}}`
   ומחזירה לי בסיום סיכום + רשימת כל ה-placeholders.
3. **מפעיל את יובל** עם ה-prompts מתוך ה-placeholders — תמונה לכל אחד.
4. **משלב את התמונות** בקבצי ה-MD וה-HTML של יעל: מחליף כל placeholder
   בהפניה לתמונה שיובל יצר ב-`yuval/outputs/`.
5. **שומר את הגרסה הסופית** (עם התמונות המשולבות) ב-`Output/`.

## זרימה מלאה — תוכן חדש מהאינטרנט (חן → יעל → יובל)

כשמתקבלת בקשה ליצירת תוכן חדש שמבוסס על מקור מהרשת:

a. **מפעיל את חן** למצוא מקור איכותי לפי הנושא / מילות המפתח.
b. **חן מחזירה** קובץ ב-`Content/` (עם לינק למקור בראשו) + דיווח קצר ולינק.
c. **אם הבקשה כללה גם שכתוב/פרסום** — ממשיך אוטומטית:
   - **מפעיל את יעל** על הקובץ שחן יצרה.
   - אם יעל ביקשה תמונות (placeholders מסוג `{{IMAGE_NEEDED}}`) —
     **מפעיל את יובל**, ומשלב הכל ב-`Output/` (לפי התהליך שלמעלה).
d. **אם הבקשה הייתה רק "מצא לי מאמר"** — עוצר אחרי שלב b ומחזיר למשתמש
   את הקובץ והלינק, בלי להפעיל את יעל.

## כללי עבודה קבועים

### Vault (זיכרון ארוך-טווח)
בתחילת **כל סשן וכל משימה** — להפעיל את הסקיל `obsidian-vault-workflow`:
לקרוא את `vault/Meeting Notes/_index.md`, לאתר את קובץ הנושא הרלוונטי ולקרוא אותו,
ובסיום המשימה לעדכן את ה-vault (Session Log + Open Questions) לפי הסקיל.
תיעוד ברמת הקובץ הבודד נמצא ב-`vault/File Docs/`.

### Git — קומיט אוטומטי
בסיום **כל משימה** שמשנה קבצים — בלי שהמשתמש צריך לבקש:
1. לסרוק את ה-staged changes לסודות ו-PII ‏(.env / מפתחות / סיסמאות / נתוני לקוחות).
2. `git add` + `commit` עם הודעה תמציתית באנגלית.
3. `git push` ל-origin: https://github.com/eladsrdp/elads
אם נמצא סוד או PII ב-staged — לעצור, להתריע למשתמש, ולא לדחוף עד שמטופל.
